# Contributing

- Create a feature branch and keep each change narrowly scoped.
- Do not modify accepted final-test metrics, tables, figures, predictions, authorization records, run state, or sentinels.
- Never use center 2 for model development, calibration fitting, threshold selection, or post-test selection.
- New model work requires a new external dataset or a newly reserved evaluation source; it must not reopen the accepted one-shot result.
- Run relevant static, portability, size, and secret scans before a pull request.
- Do not commit datasets, source patches, caches, checkpoints, raw predictions, logits, credentials, certificates, environment files, or private contact documents.
- Preserve unfavorable findings and avoid clinical-readiness or universal model-ranking claims.

No contribution is licensed for reuse until an explicit repository license is selected.
